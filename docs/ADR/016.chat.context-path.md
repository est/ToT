## Context

ADR-010 定义了 focus 指针，但没定"给 LLM 看到的上下文具体包含什么"。

候选方案：

- A：整棵树的所有消息（全量）
- B：从根到 focus 节点的路径消息（主线 + 当前分支）
- C：路径消息 + 兄弟分支的摘要（路径 + 邻居上下文）
- D：路径消息 + 用户选择的相关分支

## Decision

采用方案 B：只给 LLM 看从根到 focus 节点路径上的消息。

实现：`buildContext()` 函数遍历 focus 节点的 `parents` 字段（前缀路径），按顺序取出每个节点的 user+assistant 消息，拼成线性数组。

不看兄弟分支。不看未探索的候选分支。

## Consequences

优点：

- 上下文精简，token 用量可控
- 实现简单，parents 路径遍历即可
- 分支之间完全隔离，不会互相污染

缺点：

- LLM 不知道"旁边还有哪些分支已探索过"
- 如果用户想引用另一分支的结论，需要手动在消息里说明
- 后续可升级到方案 C（加摘要），但当前 MVP 不需要
