## Context

需要用户认证系统。候选：

- 传统密码登录（email + password）
- OAuth（Google/GitHub 等第三方）
- Passkey（WebAuthn，FIDO2）
- Magic link（邮件链接）

要求：
- 用户输入 email 作为用户名
- 不依赖第三方服务
- 安全性高于密码

## Decision

采用 Passkey（WebAuthn）。使用 `@simplewebauthn/server` 库。

RP ID 配置：
- 生产环境：`chats.est.im`
- 本地开发：当前 host（`localhost`）

## Consequences

优点：

- 无密码泄露风险
- 不需要邮件服务（magic link 需要）
- 防钓鱼（origin 绑定）
- 用户体验好（生物识别/安全密钥）

缺点：

- 浏览器兼容性（2026 年已广泛支持）
- 需要 HTTPS（生产环境）
- 用户首次需要注册 Passkey
