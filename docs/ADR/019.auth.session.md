## Context

需要 session 管理。候选：

- 独立 sessions 表
- 存在 users 行内 JSON 字段
- JWT（无状态，cookie 存 token）

要求：
- 支持多设备同时登录
- 简单实现，不引入额外表

## Decision

Session 存在 `users.sessions` JSON 字段内。

结构：`{ "random_token_hex": { "expires_at": sec, "device": "" } }`

- 每次登录生成新的 16 字节随机 token，加入 JSON
- 过期 token 在下次查询时清理
- 登出删除对应 token key

## Consequences

优点：

- 无额外表
- 多设备登录（多个 token）
- 实现简单（JSON parse/update）

缺点：

- 并发写入可能丢失（D1 单写，实际无问题）
- 清理过期 token 是懒加载（不影响正确性）
- sessions JSON 随设备增多会变大（实际可控）
